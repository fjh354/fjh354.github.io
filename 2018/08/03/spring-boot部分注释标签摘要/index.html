spring boot部分注释标签摘要

发表于 2018-08-03 | 更新于 2018-08-07

@RestController:
@RestController注解相当于@ResponseBody ＋ @Controller合在一起的作用。因此@Controller要么跳另一个controller要么跳转至页面，@RestController返回的是JSON之类的，就是AJAX跳转controller时的用法，只不过不用写@ResponseBody了而已

@Configuration
被此注解标示的类相当于XML文件内的bean标签，配置spring容器
@Configuration 中所有带 @Bean 注解的方法都会被动态代理，因此调用该方法返回的都是同一个实例。
配置需求：
1、@Configuration不可以是final类型；
2、@Configuration不可以是匿名类；
3、嵌套的configuration必须是静态类。
该注释详解：https://www.cnblogs.com/duanxz/p/7493276.html

@Scope(“prototype”):
表示每次接受一个请求创建一个action对象，相当于原xml文件内的scope属性,饿汉式懒汉式的区别

@Bean
标注在方法上(返回某个实例的方法)，等价于spring的xml配置文件中的，作用为：注册bean对象
@Bean 支持两种属性，即 initMethod 和destroyMethod,写法为initMethod =”该类的方法的方法名”,前者在创建时启动，后者在销毁时启动

@Component
也是注册bean的注解，不过可以通过配置使之自动实例，配置方法：配置@ComponentScan(basePackages = “包的路径”)，@ComponentScan如名，扫描@Component的扫描器

@ImportResource(“classpath:applicationContext-configuration.xml”)
@Import(TestConfiguration.class)
引入文件

@Value(“${相关配置文件属性名}”)
可以给该标签底下的属性赋值，赋的值为@value里引用的配置文件属性的值

@Entity
对实体注释。任何Hibernate映射对象都要有这个注释

@Table
声明此对象映射到数据库的数据表，通过它可以为实体指定表(talbe),目录(Catalog)和schema的名字。该注释不是必须的，如果没有则系统使用默认值(实体的短类名)。

@Version
该注释可用于在实体Bean中添加乐观锁支持。

@Id
声明此属性为主键。该属性值可以通过应该自身创建，但是Hibernate推荐通过Hibernate生成

@GeneratedValue
指定主键的生成策略。有如下四个值
TABLE：使用表保存id值
IDENTITY：identitycolunm
SEQUENCR ：sequence
AUTO：根据数据库的不同使用上面三个

@Column
声明该属性与数据库字段的映射关系。
注意：
1、 当POJO有属性不需要映射的时候一定要用@Transitent修饰，该注释表示此属性与表没有映射关系，

只是一个暂时的属性。
2、 @Lob注释表示该属性持久化为Blob或者Clob类型，具体取决于属性的类型。

@Repository
该标签指明此类为DAO类,

@Entity开始直至最后结尾的注释来自：https://www.cnblogs.com/hoojjack/p/6568920.html

在@Configuration下加入@EnableTransactionManagement使得支持事务
