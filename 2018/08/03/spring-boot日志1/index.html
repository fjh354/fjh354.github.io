spring boot日志1

发表于 2018-08-03 | 更新于 2018-09-03

今天根据要求学习spring boot，大概的部署都结束了，来此记录一下出现的问题

spring boot是用来干什么的？
通常在使用spring的时候需要配置一大堆的东西，现在这一些都不用自己配了，而是交给官方来做。

简单来说就是通过spring boot的生成网址自动生成带有配置文件的项目，导入eclipse以后通过eclipse的编译自动生成包结构等，也就是说spring不用再自己配置了，而是交给官方做（NM必须的，培训的时候就觉得这个配置文件我真是CNM了，抱歉爆粗口了），之后使用者可以根据自己的实际需求增删相关配置

当导入maven项目后，eclipse经过编译，包结构会出现以下三种：
src/main/java 程序开发以及主程序入口
src/main/resources 配置文件
src/test/java 测试程序

之后运行XXXapplication.java文件开启spring boot

3.启动application.java的时候报错
报错信息：org.apache.catalina.LifecycleException: Failed to start component [Connector[HTTP/1.1-8080]]
我这里是因为端口占用，因为我开着webstorm占着8080端口

4.成功启动application后，网页报错
报错信息：This application has no explicit mapping for /error, so you are seeing this as a fallback.
原因1:
Application启动类的位置不对.要将Application类放在最外侧,即包含所有子包
spring-boot会自动加载启动类所在包下及其子包下的所有组件.

原因2:
在springboot的配置文件:application.yml或application.properties中关于视图解析器的配置问题:
当pom文件下的spring-boot-starter-paren版本高时使用:
spring.mvc.view.prefix/spring.mvc.view.suffix
当pom文件下的spring-boot-starter-paren版本低时使用:
spring.view.prefix/spring.view.suffix

原因3:
控制器的URL路径书写问题
@RequestMapping(“xxxxxxxxxxxxxx”)
实际访问的路径与”xxx”不符合.

原因4:
为扫描到templates下的静态文件,添加spring-boot-starter-thymeleaf依赖就可解决
