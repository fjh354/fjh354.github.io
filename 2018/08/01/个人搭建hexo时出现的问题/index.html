个人搭建hexo时出现的问题

发表于 2018-08-01

主要问题：
1.Please make sure you have the correct access rights
and the repository exists.
2.the remote end hung up unexpectedly

首先根据官方文档hexo.io/zh-cn/docs/搭建完基本环境，问题出在部署的时候。

部署对象：github

问题1：please……….
解决：_config文件将HTTP的方式转换成ssh,增加了ssh key（方法网上一大堆）,切换到hexo文件夹下，ssh -T git@github.com ，不知道为什么基本每次部署都让我ssh一下，难道是因为我关闭了git bash?完了以后上传提交，如果不开VPN并且在大中华土地上的人几乎都会遇到问题2

问题2：the remote…
解决：其实就是网络不行，修改最低速度限制和时间：
git config –global http.lowSpeedLimit 0
git config –global http.lowSpeedTime 999999 //6个9是最大值，开发者喜欢这个姿势？

修改上传时HTTP的缓存:
git config –global http.postBuffer 524288000 //500M,想要更多可以自己乘，我乘三倍next主题的hexo才传上去

基本上就没什么问题了，接下来是几个细节：
细节：GitHub Repository name为 用户名.github.io不然不能直接用XXX.github.io进入网站，而是需要一个全称
细节2：如果依靠SSH方式部署，那么在生成的ssh key填入github的框内时，需要勾选读写权限
细节3：hexo配置文件内 属性：后必须带空格，比如 repo:空格xxxxxxx
